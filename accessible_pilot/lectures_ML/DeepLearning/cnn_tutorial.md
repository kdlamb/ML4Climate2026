# Tutorial on Global Temperature Trends with Deep Learning

## Predicting global temperature from greenhouse gas concentrations

This hands-on tutorial accompanies the [Convolutional Neural Networks](cnn.md) notes.
We use deep learning to predict global surface temperature from the global atmospheric
concentrations of CO₂ and CH₄. First we build a **dense neural network (NN)** that
predicts temperature everywhere on Earth as one long vector; then we build a
**convolutional neural network (CNN)** that predicts the temperature **map** directly,
keeping its spatial structure. It is based on notebooks by Weiwei Zhan and Francesco
Immorlano developed for a [LEAP](https://leap.columbia.edu/) bootcamp
([NN example](https://github.com/leap-stc/LEAP-bootcamps/blob/main/Codes/Day2/Neural_Network_example.ipynb),
[CNN example](https://github.com/leap-stc/LEAP-bootcamps/blob/main/Codes/Day2/CNN_example.ipynb)).

```python
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras.models import Model, load_model

cwd = os.getcwd()
```

## The ClimateBench data

**ClimateBench** is a spatial–temporal dataset of simulations generated by the
**NorESM2** climate model. It provides both **historical** simulations and **future
projections** under different **SSP (Shared Socioeconomic Pathway)** scenarios, which
make different assumptions about future anthropogenic emissions. The **inputs** are
greenhouse-gas concentrations (CO₂, CH₄); the **outputs** are global surface-temperature
maps (`tas`) on a 96 (latitude) × 144 (longitude) grid.

```python
def open_dataset(file_path):
    """Open a ClimateBench file. For a gs:// url, read the matching .zarr store."""
    if 'gs://' in file_path:
        return xr.open_dataset(f"{file_path}.zarr", engine='zarr')
    ds = xr.open_dataset(f"{file_path}.nc")
    ds.attrs['file_name']
    return ds

scenarios = ['historical', 'ssp126', 'ssp370', 'ssp585']
inputs  = [os.path.join(train_path, f"inputs_{s}")  for s in scenarios]
inputs.append(os.path.join(test_path, "inputs_ssp245"))
inputs.sort(key=lambda x: x.split('_')[-1])

outputs = [os.path.join(train_path, f"outputs_{s}") for s in scenarios]
outputs.append(os.path.join(test_path, "outputs_ssp245"))
outputs.sort(key=lambda x: x.split('_')[-1])
```

### The emissions scenarios (inputs)

```python
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
colors = ['tab:blue', 'tab:green', 'tab:purple', 'tab:orange', 'tab:red']
for i, inp in enumerate(inputs):
    label = inp.split('_')[-1]
    X = open_dataset(inp)
    X['CO2'].plot(label=label, color=colors[i], linewidth=2, ax=axes[0])
    X['CH4'].plot(label=label, color=colors[i], linewidth=2, ax=axes[1])
axes[0].set_ylabel("Cumulative anthropogenic CO2\nemissions since 1850 (GtCO2)")
axes[1].set_ylabel("Anthropogenic CH4\nemissions (GtCH4 / year)")
axes[0].set_title('CO2'); axes[1].set_title('CH4'); axes[0].legend()
plt.tight_layout()
```

**What the plot shows.** Two line panels sharing a time axis from 1850 to 2100, one
line per scenario:

- **Left (CO₂)**: cumulative anthropogenic CO₂ since 1850, in gigatonnes. All scenarios
  share the same rising **historical** curve up to about 2025, then fan out.
  **ssp585** (the "business as usual" high-emissions path) climbs steepest; **ssp126**
  (optimistic) bends over and flattens as emissions are brought down by 2100; **ssp245**
  and **ssp370** sit in between.
- **Right (CH₄)**: methane emissions over the same period, with a similar but more
  complicated fan of scenarios.

We roughly sit between the middle pathways (ssp245/ssp370) today.

(with a wide band), while **ssp126** stays lowest, roughly holding to the ~1.5 °C Paris

future projections.

## Part 1: A dense neural network

### Prepare the data

We train on **three historical variants plus three future scenarios** and **test on
ssp245** (a middle scenario, so the network *interpolates* rather than *extrapolates*).
For the dense NN the target map is **flattened** into one long vector of 96 × 144 =
13,824 values.

```python
def prepare_predictor(data_sets, data_path, time_reindex=True):
    if isinstance(data_sets, str): data_sets = [data_sets]
    X_all = []
    for file in data_sets:
        X_all.append(open_dataset(f"{data_path}inputs_{file}"))
    X = xr.concat(X_all, dim='time')
    if time_reindex:
        X = X.assign_coords(time=np.arange(len(X.time)))
    return X, None

def prepare_predictand(data_sets, data_path, time_reindex=True):
    if isinstance(data_sets, str): data_sets = [data_sets]
    Y_all = []
    for file in data_sets:
        Y_all.append(open_dataset(f"{data_path}outputs_{file}"))
    Y = xr.concat(Y_all, dim='time').mean('member')
    Y = Y.rename({'lon': 'longitude', 'lat': 'latitude'}).transpose(
        'time', 'latitude', 'longitude').drop_vars(['quantile'])
    if time_reindex:
        Y = Y.assign_coords(time=np.arange(len(Y.time)))
    return Y, None

train_files = ["historical", "ssp585", "ssp126", "ssp370", "hist-aer", "hist-GHG"]
X_train_xr, _ = prepare_predictor(train_files, train_path)
y_train_xr, _ = prepare_predictand(train_files, train_path)
X_test_xr,  _ = prepare_predictor('ssp245', data_path=test_path, time_reindex=False)
y_test_xr,  _ = prepare_predictand('ssp245', data_path=test_path, time_reindex=False)

X_train_df = pd.DataFrame({"CO2": X_train_xr["CO2"].data, "CH4": X_train_xr["CH4"].data},
                          index=X_train_xr["CO2"].coords['time'].data)
X_test_df  = pd.DataFrame({"CO2": X_test_xr["CO2"].data,  "CH4": X_test_xr["CH4"].data},
                          index=X_test_xr["CO2"].coords['time'].data)
y_train_df = pd.DataFrame(y_train_xr["tas"].stack(z=("latitude", "longitude")).to_pandas())

# Standardize the inputs
mean, std = X_train_df.mean(), X_train_df.std()
X_train_df = (X_train_df - mean) / std
X_test_df  = (X_test_df  - mean) / std
X_train, y_train, X_test = X_train_df.to_numpy(), y_train_df.to_numpy(), X_test_df.to_numpy()
print(X_train.shape, y_train.shape)
```

**What the numbers show.** `(753, 2) (753, 13824)` — **753** monthly samples, each with
just **2** input features (CO₂, CH₄), mapped to **13,824** output pixels (96 × 144).
This is a deliberately hard problem: two numbers in, a whole global map out, from only
~750 examples. The single biggest limitation of ML for problems like this is that we
**do not have much data**.

### Define, train, and save the network

Three hidden layers of 64 neurons with ReLU, and a **linear** output layer of 13,824
units (one per pixel).

```python
n_neuron, activation = 64, 'relu'
num_epochs, learning_rate, minibatch_size = 50, 0.001, 64

tf.keras.backend.clear_session()   # forget any layers left in memory
model = tf.keras.Sequential([
    tf.keras.Input(shape=(X_train.shape[1],)),
    tf.keras.layers.Dense(64, activation="relu", name="hidden_layer_1"),
    tf.keras.layers.Dense(64, activation="relu", name="hidden_layer_2"),
    tf.keras.layers.Dense(64, activation="relu", name="hidden_layer_3"),
    tf.keras.layers.Dense(y_train.shape[1], activation='linear', name="output_layer"),
])
model.summary()
model.compile(loss='mse', optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate))
```

**What `model.summary()` shows.** A table of layers and parameter counts. The three
hidden layers are small (2→64 is 192 parameters; each 64→64 is 4,160), but the
**output layer dominates**: 64 × 13,824 + 13,824 = **898,560** parameters. The whole
network has **about 907,000 trainable parameters** — nearly a million, almost all in
that final wide layer.

```python
early_stop = keras.callbacks.EarlyStopping(monitor='val_loss', patience=20)
history = model.fit(X_train, y_train, batch_size=minibatch_size, epochs=num_epochs,
                    validation_split=0.2, verbose=1, callbacks=[early_stop])

model_path = os.path.join(cwd, 'saved_model')
os.makedirs(model_path, exist_ok=True)
model.save(os.path.join(model_path, 'NN_model.keras'))
```

We hold out **20%** of the training data as a validation set (`validation_split=0.2`)
and use **early stopping**: training halts once the validation loss stops falling for
20 epochs, rather than always running all 50. On this small dataset each epoch trains
in **well under a second**. Saving the trained model lets us reload it later without
retraining.

```python
plt.figure()
plt.plot(history.history['loss'], label='training loss')
plt.plot(history.history['val_loss'], label='validation loss')
plt.xlabel('epoch'); plt.ylabel('MSE loss'); plt.legend()
```

**What the plot shows.** Training and validation **mean-squared-error** loss versus
epoch. Both drop sharply in the first ~25 epochs and then flatten; early stopping ends
the run once the validation loss stops improving. (The validation loss can sit slightly
*below* the training loss early on, a quirk of when each is measured within an epoch.)

We reshape the flat 13,824-vector prediction back into a 96 × 144 map so we can compare
it to the truth. Inference is **near-instant** — the whole value of an emulator.


**What the plot shows.** A 3-row, 2-column grid of temperature maps: **left column =
NN prediction**, **right column = ssp245 truth**, for **2030, 2050, 2100** (top to
bottom). In each row the prediction and truth look broadly alike — the same large-scale
pattern of warming, strongest in the **Arctic**, deepening from 2030 to 2100 — but the
prediction misses fine detail. The NN captures the structure without reproducing every
regional feature.

```python
lat, lon = 40.7128, -74.0060 % 360     # New York City
fig, ax = plt.subplots(figsize=(9, 4))
y_test_xr.sel(latitude=lat, longitude=lon, method='nearest').tas.plot(
    marker='o', ax=ax, label='truth')
y_test_pre.sel(latitude=lat, longitude=lon, method='nearest').tas.plot(
    marker='o', ax=ax, label='prediction')
ax.legend(); ax.set_ylabel('temperature (°C)')
plt.tight_layout()
```

**What the plot shows.** Two temperature time series for the grid cell nearest **New
York City**, from 2015 to 2100: the **truth** and the **NN prediction**. Both rise with
the same overall **trend**, but the prediction is **much smoother** — it tracks the mean
warming while flattening out the year-to-year wiggles. This is a general habit of neural
networks trained on a mean-squared-error loss: to minimize average error they **predict
toward the middle** and tend to **underestimate extremes** unless training is designed
to preserve them.

```python
def global_mean_std_plot(X, label, color, ax, var='tas'):
    weights  = np.cos(np.deg2rad(X.latitude))
    tas_mean = X[var].weighted(weights).mean(['latitude', 'longitude']).data
    tas_std  = X[var].weighted(weights).std(['latitude', 'longitude']).data
    ax.plot(X.time.data, tas_mean, label=label, color=color, linewidth=2)
    ax.fill_between(X.time.data, tas_mean + tas_std, tas_mean - tas_std,
                    facecolor=color, alpha=0.2)

fig, ax = plt.subplots(figsize=(9, 4))
global_mean_std_plot(y_test_xr,  label='truth',      ax=ax, color='tab:blue')
global_mean_std_plot(y_test_pre, label='prediction', ax=ax, color='tab:orange')
ax.set_xlabel('time'); ax.set_ylabel('global mean temperature (°C)'); ax.legend()
plt.tight_layout()
```

**What the plot shows.** Area-weighted **global-mean** temperature versus year, truth vs.
prediction, each with a ± one-standard-deviation band. The prediction reproduces both the
**rising mean** and a good deal of the **spread** (band width); the truth is a little
noisier year to year, but the emulator gets the envelope right.

## Part 2: A convolutional neural network

Now we predict the temperature **map** directly with a CNN. The key difference from the
dense NN: **no flattening** — the target stays a 96 × 144 image.

```python
X_train_df = pd.DataFrame({"CO2": X_train_xr["CO2"].data, "CH4": X_train_xr["CH4"].data},
                          index=X_train_xr["CO2"].coords['time'].data)
X_test_df  = pd.DataFrame({"CO2": X_test_xr["CO2"].data,  "CH4": X_test_xr["CH4"].data},
                          index=X_test_xr["CO2"].coords['time'].data)
y_train = y_train_xr['tas'].data          # keep the 2-D map, do NOT flatten
y_test  = y_test_xr['tas'].data

mean, std = X_train_df.mean(), X_train_df.std()
X_train = ((X_train_df - mean) / std).to_numpy()
X_test  = ((X_test_df  - mean) / std).to_numpy()
print(X_train.shape, y_train.shape, X_test.shape, y_test.shape)
```

**What the numbers show.** `(753, 2) (753, 96, 144) (86, 2) (86, 96, 144)` — the same
753 training and 86 test samples with 2 inputs, but now the targets keep their **2-D map
shape (96 × 144)** instead of being flattened to 13,824.

### Define the CNN architecture

Because we go from **2 numbers up to a 96 × 144 map**, the CNN is an **up-sampling**
network. A dense layer expands the 2 inputs to 6,912 values, reshaped to a 12 × 18 × 32
feature block; three **transposed convolutions with stride 2** each **double** the
spatial size (12×18 → 24×36 → 48×72 → 96×144); a final transposed convolution collapses
the 32 filters to the **single** temperature channel.

```python
n_filters, n_neurons, activation = 32, 32, 'relu'
kernel_size, learning_rate, minibatch_size, num_epochs = 4, 0.001, 64, 100

model = Sequential()
model.add(Input(shape=(X_train.shape[1],)))
model.add(Dense(n_filters * 12 * 18, activation=activation))   # -> 6912
model.add(Reshape((12, 18, n_filters)))                        # -> 12 x 18 x 32
model.add(Conv2DTranspose(n_filters, kernel_size, activation=activation,
                          strides=2, padding='same'))          # -> 24 x 36 x 32
model.add(Conv2DTranspose(n_filters, kernel_size, activation=activation,
                          strides=2, padding='same'))          # -> 48 x 72 x 32
model.add(Conv2DTranspose(n_filters, kernel_size, activation=activation,
                          strides=2, padding='same'))          # -> 96 x 144 x 32
model.add(Conv2DTranspose(1, kernel_size, activation="linear", padding="same"))  # -> 96 x 144 x 1
model.summary()
model.compile(loss='mse', optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate))
```

**What `model.summary()` shows.** The dense expansion layer holds 2 × 6,912 + 6,912 =
**20,736** parameters; each of the three 32→32 transposed convolutions (a 4 × 4 kernel)
holds 4 × 4 × 32 × 32 + 32 = **16,416**; the final 32→1 layer holds **513**. That totals
**about 70,000 trainable parameters** — an order of magnitude **fewer than the dense
NN's ~907,000**, because weight sharing means the same small kernels are reused across
the whole map instead of learning an independent weight for every one of the 13,824
output pixels.

### Train, save, and evaluate

```python
early_stop = keras.callbacks.EarlyStopping(monitor='val_loss', patience=20)
history_cnn = model.fit(X_train, y_train, batch_size=minibatch_size, epochs=num_epochs,
                        validation_split=0.2, verbose=1, callbacks=[early_stop])
model.save(os.path.join(model_path, 'CNN_model.keras'))
```

Each CNN epoch takes **about 12 seconds** here (versus under a second for the dense NN),
because the transposed convolutions do far more arithmetic. In practice you would train
a CNN on a **GPU**, where it runs much faster; this run used a CPU.

Comparing the two models' loss curves: the CNN's **training** loss is slightly *higher*
than the dense NN's, but its **validation** loss is slightly *lower* — hinting that the
CNN **generalizes a little better**, and it does so with **far fewer weights**. In
general we would expect a CNN to have the edge on a spatial task like this (its spatial
inductive bias), though here the two are close and the result is problem-dependent.

```python
model = load_model(os.path.join(model_path, 'CNN_model.keras'))
```

**What the plot shows.** The same 3×2 map grid as before, now with the **CNN**
prediction on the left. It again captures the right large-scale warming pattern
(Arctic-amplified, growing to 2100), but looks **blurrier / lower-resolution** than the
truth in the details — a sign the architecture could be tuned further, and that we are
**data-limited** with only 753 training samples.

## Wrap-up

We predicted global surface temperature from CO₂ and CH₄ two ways:

- A **dense neural network** flattens the target to a 13,824-vector and needs **~907,000
  parameters**, almost all in one wide output layer.
- A **CNN** predicts the 96 × 144 map **directly** through up-sampling transposed
  convolutions, using **~70,000 parameters** — far fewer, thanks to weight sharing — and
  generalizes at least as well.

Both emulators reproduce the large-scale warming pattern and the global-mean trend, both
**smooth away interannual variability and extremes** (a consequence of the MSE loss), and
both are held back by having only ~750 training samples — a recurring reality of applying
deep learning to climate simulations.

## Try it yourself

1. **Set the random seed.** These models randomly initialize their weights (and SGD
   samples random batches), so results shift slightly run to run. Add
   `tf.random.set_seed(42)` before building each model and confirm the losses reproduce.
2. **Test on a different scenario.** Retrain on the other scenarios and test on
   **ssp585** instead of ssp245. Does the emulator do worse when it has to *extrapolate*
   to a hotter scenario than any it trained on, rather than *interpolate*?
3. **Tune the CNN.** Try a larger `kernel_size`, more `n_filters`, or more up-sampling
   blocks. Can you sharpen the blurry maps without overfitting the small dataset?
4. **Add a metric.** Pass `metrics=['RootMeanSquaredError']` to `model.compile` and watch
   the RMSE (in °C) alongside the MSE loss during training.

The last two run in seconds and neither retrains a model, so they are good to try in
class if there is time.

5. **Read off the architecture, no training.** Rebuild *only* the CNN `Sequential(...)`
   block with a different `kernel_size` (try 3 and 8) or a different `n_filters`, then
   call `model.summary()`. Do **not** call `model.fit`. The output stays 96 × 144
   whatever the kernel size: why? (Look at `padding='same'` and the three stride-2
