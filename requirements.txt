# Needed by the lecture-note pages that re-execute during the build
# (classification, regression, treesforests, dimensionalityreduction, clustering).
# They build every figure from scikit-learn's bundled toy datasets, so no network
# access is required. Notebooks needing heavier deps (xarray, kagglehub) are
# excluded from execution in _config.yml and ship their saved outputs instead.
#
# scikit-learn is PINNED, on purpose. CI re-executes these notebooks and publishes
# whatever comes out, and `allow_errors: true` means a version incompatibility gets
# rendered onto the live page as a traceback rather than failing the build. The pages
# also quote exact numbers in their prose (coefficients, ARI scores, AUCs), which a
# version bump can silently shift. This pin is the version those numbers were checked
# against. Bumping it is fine — but re-run the notebooks and re-check the numbers in
# the prose against the new output, don't just change it and push.
#
# numpy is capped because scikit-learn 1.3.2's wheels are built against the numpy 1.x
# ABI; pip would otherwise install numpy 2.x and break it at import.
numpy<2
scikit-learn==1.3.2
pandas
